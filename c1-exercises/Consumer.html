    axis.set_xlabel(xlabel)
    axis.set_ylabel(ylabel)

    for msg in kc:
        t += 1
        j = json.loads(msg.value)

        k = j["authzn_amt"]

# for each message, grab its value
lines = kvs.map(lambda x: x[1])

# split the line into its words, counting each word

    .map(lambda word: (word, 1)) \
    .reduceByKey(lambda a, b: a+b)

Exercise 5 - Polling Kafka with the Confluent Kafka library for Python¶

Next, let's repeat the first exercise, but this time, we'll use the Kafka library from Confluent. Confluent is a company run by Jay Krebs, who led the team that originall created Kafka while he worked at LinkedIn. This library has much of the same functionality as the Kafka-python library we used earlier, as we'll see below.

In [ ]:
from confluent_kafka import Consumer, KafkaError
import json

lst_brokers = "localhost:9092"
topic = "topic01"

c = Consumer({'bootstrap.servers': lst_brokers, 'group.id': 'mygroup',
            'default.topic.config': {'auto.offset.reset': 'largest'}})
c.subscribe([topic])
running = True
while running:
    msg = c.poll()
    if not msg.error():
        try:
            j = json.loads(msg.value().decode('utf-8'))
            print("auth_id:", j["auth_id"], "acct_avl_cash_before_amt:", j["acct_avl_cash_before_amt"])
            print("-" * 100)
        except Exception as e:
            print(str(e))
    elif msg.error().code() != KafkaError._PARTITION_EOF:
        print(msg.error())
        running = False
c.close()

One not insignificant detail is that the defaults between the kafka-python and confluent-kafka-python implementations have different defaults. In the latter, we must pass the auto.offset.reset parameter in the default.topic.config section of the consumer creation code and set it to largest to mimic the default functionality of the kafka-python implementation. Note that while this is a non-production-ready example, the concept of default values is something that should always be considered when implementing a solution.

Also note that with the confluent-kafka-python implementation, there may be a significant delay between when the consumer is created and when the first messages are displayed.

This script will start a window multiplexer called 'tmux' with two Terminal panes side by side. The pane on the left runs the Netcat program and opens port 9999. Any text typed into this pane will be sent to port 9999. At the same time, the right pane runs a network wordcount Python script listening on port 9999. The right window will display a timestamp every five seconds, along with an aggregate, non-accumulating word count of words it has received from port 9999 in the last five seconds.

About five seconds after you enter the tmux environment, hit Ctrl-b, then the 'o' key to swap focus to the left-hand pane and start entering text. Note that you will need to hit the Enter key to send the text to port 9999.

To exit this environment, hit Ctrl-c twice while the left pane has focus.
